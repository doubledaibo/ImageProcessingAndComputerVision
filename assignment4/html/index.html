Introduction

In this project we've implemented HOG(histogram of oriented gradient)-based pedestrian detector.

Algorithm Detail

  • Compute hog feature vectors for training data
  • Learn a classifier using SVM
  • Detect candidates on an image using sliding-window mechanism with different scales
  • Post-process candidates using non-maximum suppression

Task Finished

  • Basic Framework
  • Tried different normalization mechanisms
  • Self-define non-maximum suppression: change the original comparision to IoU (intersection over union)

Setting

  • number pixels per cell: 8 x 8
  • number cells per block: 2 x 2 (50% overlap between neighbor blocks)
  • number of bins: 9
  • normalization mechanism: L1-norm
  • scales: 0.25, 0.3, 0.5, 1
  • thresholds: 3.5, 4.5, 6.5, 6.5
  • non-maximum suppression threshold: 0.3

Analysis

  • The classifier is not trained very well (see performance on validation set below) even using opencv's hog extractor. Maybe there are bugs in detection phase, or linear classifier is not good enough.
